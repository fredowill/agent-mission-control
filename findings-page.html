Architecture PID-based liveness replaced broken process-count heuristic

The original Mission Control counted all claude.exe processes globally and assigned "active" to the top N by recency. This was fundamentally broken — no per-session correlation, cross-project contamination. Replaced with per-session claudePid tracking: hook walks process tree on first fire, server checks if PID is alive. Terminal open = active.

Sessions: ed2482, c88d3c, 2a35e8 • Resolved 2026-03-03
Architecture Session IDs change within the same terminal window

Claude Code generates new session IDs on /plan → implement transitions and likely on /compact. The claude.exe process (PID) stays the same. This caused duplicate cards in the dashboard. Fixed by merging sessions that share a claudePid into a single card with combined prompts and inherited missions.

Discovered during PID implementation • Resolved 2026-03-03
Architecture Windows process tree is walkable (contrary to initial analysis)

The critic agent concluded process.ppid was "useless on Windows." Testing proved the tree is claude.exe → bash → bash → bash → node hook.js. Walking up 3 levels via PowerShell Get-CimInstance Win32_Process reliably finds claude.exe. The lesson: always test OS assumptions empirically.
